1. WHAT IS SPRING & SPRING BOOT?

What is Spring?

  • Spring is a lightweight Java framework.
  • It provides a comprehensive programming and configuration model for modern Java-based enterprise applications.
  • It is built on core concepts like IoC, DI and AOP.

What is Spring Boot?

  • Spring Boot is an extension of the Spring framework.
  • It simplifies development by auto-configuration and convention over configuration.
  • It helps to create stand-alone, production-ready applications with minimal setup.
  • It has embedded servers (Tomcat, Jetty, Undertow) and reduces boilerplate code.
Why Spring Boot?
  • Faster development
  • Auto configuration
  • Embedded servers
  • Production ready
  • Less XML, more Annotations
  • Easy Dependency Management
  • Built-in Actuator & Monitoring

3. IoC CONTAINER & DEPENDENCY INJECTION

3.1 What is IoC?

  • IoC (Inversion of Control) is a principle where the control of object creation and management is transferred from application code to the Spring container.
  • The container creates objects, wires them together, manages their lifecycle and dependencies.

3.2 Spring IoC Container

  • Spring IoC container is responsible for creating, configuring and assembling the beans.
  • The container provides the beans to the application when required.

3.3 Dependency Injection (DI)

  • DI is a design pattern where the container injects the required dependencies into the objects.
  • It helps in achieving loose coupling and easy unit testing.
Types of DI
  • Constructor Injection (recommended)
  • Setter Injection
  • Field Injection

Example (Constructor Injection)

public class UserService {
    private UserRepository userRepository;

    public UserService(UserRepository repo) {
        this.userRepository = repo;
    }
}

3.5 Key Points

  • We just declare our dependencies, container takes care of creation and injection.

6.3 Example — Simple REST Controller

@RestController
@RequestMapping("/api/users")
public class UserController {

    @GetMapping("/{id}")
    public User getUser(@PathVariable Long id) {
        return new User(id, "Aman", "aman@example.com");
    }

    @PostMapping
    public User createUser(@RequestBody User user) {
        user.setId(101L);
        return user;
    }

    @PutMapping("/{id}")
    public User updateUser(@PathVariable Long id,
                           @RequestBody User user) {
        user.setId(id);
        return user;
    }

    @DeleteMapping("/{id}")
    public String deleteUser(@PathVariable Long id) {
        return "User deleted with id: " + id;
    }
}
Explanation
  • @RestController = @Controller + @ResponseBody
  • @RequestMapping defines base URL for the class
  • @GetMapping handles GET request
  • @PostMapping handles POST request
  • @PutMapping handles PUT request
  • @DeleteMapping handles DELETE request
  • @PathVariable fetches value from URL
  • @RequestBody binds request body to object
Sample JSON (Request Body)
{
  "name":  "Aman",
  "email": "aman@example.com",
  "age":   25
}

11. AUTO-CONFIGURATION & STARTERS

11.1 What is Auto-Configuration?

  • Spring Boot automatically configures beans based on the dependencies (JARs) present on the classpath.
  • Example: if spring-boot-starter-web is on the classpath → Boot auto-configures DispatcherServlet, embedded Tomcat, Jackson JSON converter.
  • If H2 + spring-boot-starter-data-jpa are present → Boot auto-configures an in-memory DataSource + EntityManager.
  • Enabled by @EnableAutoConfiguration (included inside @SpringBootApplication).
Classpath JARs(starters, drivers)
→
@EnableAutoConfigurationreads AutoConfiguration.imports
→
@Conditional checks(class present? bean missing?)
→
Beans auto-registered

11.2 How it works internally

  • Auto-configuration classes are listed in META-INF/spring/org.springframework.boot.autoconfigure.AutoConfiguration.imports.
  • Each class is guarded by conditional annotations:
AnnotationBean is created only if…
@ConditionalOnClassa given class is on the classpath
@ConditionalOnMissingBeanthe user has NOT defined the bean already (your bean always wins)
@ConditionalOnPropertya property has a specific value
@ConditionalOnWebApplicationthe app is a web application

Interview line: "Auto-configuration is opinionated but overridable — Boot backs off the moment you define your own bean."

11.3 Starters

  • A starter is a curated set of compatible dependencies bundled under one artifact — no version juggling.

12. JPA & HIBERNATE (ENTITY MAPPING)

12.1 What is JPA / Hibernate?

  • JPA (Jakarta Persistence API) is a specification for ORM (Object Relational Mapping) in Java.
  • Hibernate is the most popular implementation of JPA — the default in Spring Boot.
  • ORM maps Java classes → database tables, objects → rows, fields → columns.
Relationship
JPA(specification / interface)
implemented by →
Hibernate(implementation)
used via →
Spring Data JPA(abstraction layer)

12.2 Entity Example

@Entity
@Table(name = "students")
public class Student {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(name = "full_name", nullable = false, length = 100)
    private String name;

    @Column(unique = true)
    private String email;

    private int age;
    // getters and setters
}

12.3 Key JPA Annotations

AnnotationPurpose
@EntityMarks the class as a JPA entity (a table).
@TableCustomizes table name / schema.
@IdMarks primary key field.
@GeneratedValueAuto-generates PK — IDENTITY, SEQUENCE, AUTO, TABLE.
@ColumnColumn name, nullable, unique, length.
@TransientField is NOT persisted to the database.
@Enumerated(EnumType.STRING)Store enum as string instead of ordinal.
@CreationTimestamp / @UpdateTimestampAuto audit timestamps (Hibernate).

12.4 Relationships

AnnotationMeaningExample
@OneToOneOne row ↔ one rowStudent ↔ Address
@OneToManyOne row → many rowsDepartment → Students
@ManyToOneMany rows → one row (owning side, holds FK)Student → Department
@ManyToManyMany ↔ many (join table)Student ↔ Course
@Entity
public class Student {
    @ManyToOne(fetch = FetchType.LAZY)
    @JoinColumn(name = "department_id")
    private Department department;
}

@Entity
public class Department {
    @OneToMany(mappedBy = "department", cascade = CascadeType.ALL)
    private List<Student> students;
}
FetchType — LAZY vs EAGER
  • LAZY → related data loaded only when accessed. (recommended)
  • EAGER → related data loaded immediately with the parent.
  • Defaults: @ManyToOne / @OneToOne = EAGER, @OneToMany / @ManyToMany = LAZY.
N+1 Problem (hot interview topic)

Loading N parents then firing 1 query per child collection = N+1 queries. Fix: JOIN FETCH in JPQL, @EntityGraph, or batch fetching.

12.5 ddl-auto Options

ValueBehaviour

updateAdd missing tables/columns (dev convenience).
createDrop & create schema at startup.
create-dropCreate at startup, drop at shutdown (tests).

14. DTOs, LOMBOK & MAPPING

14.1 Why DTOs?

  • DTO (Data Transfer Object) carries data between client and server — never expose the entity directly.
  • Hides internal fields (passwords, audit columns), avoids lazy-loading serialization issues, lets API shape evolve independently of DB schema.
public record StudentDto(
        Long id,
        String name,
        String email) { }   // Java record = immutable DTO

14.2 Entity ↔ DTO Mapping

// Manual mapping (simple & explicit)
public StudentDto toDto(Student s) {
    return new StudentDto(
        s.getId(), s.getName(), s.getEmail());
}

For big projects use MapStruct (compile-time, fast) or ModelMapper (reflection-based).

@Mapper(componentModel = "spring")
public interface StudentMapper {
    StudentDto toDto(Student student);
    Student toEntity(StudentDto dto);
}

14.3 Lombok — kill the boilerplate

AnnotationGenerates
@Getter / @SetterGetters / setters for all fields
@NoArgsConstructor / @AllArgsConstructorConstructors
@ToString / @EqualsAndHashCodetoString(), equals(), hashCode()
@DataAll of the above (getter+setter+toString+equals+requiredArgsCtor)
@BuilderFluent builder: Student.builder().name("Aman").build()
@Slf4jA ready log field for logging
@Entity
@Getter @Setter
@NoArgsConstructor @AllArgsConstructor
@Builder
public class Student {
    @Id @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;
    private String name;
    private String email;
}

Caution: avoid @Data on JPA entities — its equals/hashCode/toString can trigger lazy loading & recursion. Prefer @Getter @Setter.

16. EXCEPTION HANDLING

16.1 The pieces

AnnotationRole

@ResponseStatusMaps an exception class to an HTTP status.

16.2 Custom Exception + Global Handler

// 1. Custom exception
public class ResourceNotFoundException
        extends RuntimeException {
    public ResourceNotFoundException(String msg) {
        super(msg);
    }
}

// 2. Throw from service
Student s = repo.findById(id)
    .orElseThrow(() -> new ResourceNotFoundException(
        "Student not found with id: " + id));
// 3. Global handler
@RestControllerAdvice
public class GlobalExceptionHandler {

    @ExceptionHandler(ResourceNotFoundException.class)
    public ResponseEntity<ErrorResponse> handleNotFound(
            ResourceNotFoundException ex) {
        ErrorResponse err = new ErrorResponse(
            404, ex.getMessage(), LocalDateTime.now());
        return new ResponseEntity<>(err, HttpStatus.NOT_FOUND);
    }

    @ExceptionHandler(MethodArgumentNotValidException.class)
    public ResponseEntity<Map<String,String>> handleValidation(
            MethodArgumentNotValidException ex) {
        Map<String,String> errors = new HashMap<>();
        ex.getBindingResult().getFieldErrors().forEach(e ->
            errors.put(e.getField(), e.getDefaultMessage()));
        return ResponseEntity.badRequest().body(errors);
    }

    @ExceptionHandler(Exception.class)   // fallback
    public ResponseEntity<String> handleAll(Exception ex) {
        return ResponseEntity.status(500)
                             .body("Something went wrong");
    }
}

16.3 Error Response shape

{
  "status": 404,
  "message": "Student not found with id: 5",
  "timestamp": "2026-07-02T10:15:30"
}

Interview line: "One @RestControllerAdvice = single source of truth for error JSON; controllers stay clean; specific handlers first, Exception.class as fallback."

17. TRANSACTIONS (@Transactional)

17.1 What is a transaction?

  • A group of DB operations that succeed or fail as one unit (ACID: Atomicity, Consistency, Isolation, Durability).
  • @Transactional on a service method → Spring opens a transaction before the method and commits on success / rolls back on RuntimeException.
@Service
public class BankService {

    @Transactional
    public void transfer(Long from, Long to,
                         BigDecimal amount) {
        accountRepo.debit(from, amount);
        accountRepo.credit(to, amount);
        // if credit fails → debit is rolled back too
    }
}

17.2 Key attributes

AttributeMeaning
propagationREQUIRED (default: join or create), REQUIRES_NEW (always new txn), SUPPORTS, MANDATORY, NEVER…
isolationREAD_COMMITTED, REPEATABLE_READ, SERIALIZABLE…
readOnly = trueOptimization hint for pure-read methods.
rollbackForAlso roll back for checked exceptions: rollbackFor = Exception.class
timeoutFail if transaction exceeds N seconds.
Gotchas (asked a lot)
  • Rollback happens for unchecked (RuntimeException/Error) by default — not for checked exceptions.
  • Self-invocation doesn't work: calling a @Transactional method from another method of the same class bypasses the proxy → no transaction.
  • Works via AOP proxy, so the method must be public.

18.3 Modern configuration (SecurityFilterChain bean)

@Configuration
@EnableWebSecurity
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http) throws Exception {
        http
            .csrf(csrf -> csrf.disable())                     // stateless REST API
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/auth/**").permitAll()  // public: login/register
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated())                // everything else needs login
            .sessionManagement(s ->
                s.sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .httpBasic(Customizer.withDefaults());
        return http.build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder();
    }
}

18.5 JWT in one paragraph

JWT (JSON Web Token) = header.payload.signature. Login once → server returns a signed token → client sends it in Authorization: Bearer <token> on every request → a filter validates the signature and sets the SecurityContext. Stateless — no server session.

23. INTERVIEW RAPID-FIRE SHEET

QuestionOne-line answer
What does @SpringBootApplication contain?@Configuration + @EnableAutoConfiguration + @ComponentScan.
@Controller vs @RestController?RestController = Controller + ResponseBody (returns JSON, not a view).
@Component vs @Bean?@Component = class-level auto-detection; @Bean = method-level manual definition (for third-party classes).
Default bean scope?Singleton — one instance per IoC container.
Field vs Constructor injection?Constructor is preferred: immutability, mandatory deps, easy testing.
@Qualifier vs @Primary?Primary = default among candidates; Qualifier = pick a specific one at injection point.
How to change server port?server.port=9090 in application.properties.
@PathVariable vs @RequestParam?PathVariable reads from URL path (/users/5); RequestParam from query string (?id=5).
PUT vs PATCH?PUT replaces the whole resource; PATCH updates part of it.
JPA vs Hibernate?JPA is the specification; Hibernate is an implementation of it.
findById vs getReferenceById?findById hits DB, returns Optional; getReferenceById returns a lazy proxy.
What is the N+1 problem?1 query for parents + N queries for children; fix with JOIN FETCH / @EntityGraph.
When does @Transactional roll back?On unchecked exceptions by default; use rollbackFor for checked ones.
Authentication vs Authorization?Who you are vs what you're allowed to do.
How does Boot know what to auto-configure?AutoConfiguration.imports file + @Conditional checks on the classpath.
Only expose health & metrics endpoints?management.endpoints.web.exposure.include=health,metrics
How to run different configs per env?Profiles: application-dev.properties + spring.profiles.active=dev.
Embedded servers supported?Tomcat (default), Jetty, Undertow.
How to make a jar executable?mvn clean package → java -jar app.jar (Boot's fat/uber jar).
@MockBean vs @Mock?@MockBean replaces a bean inside the Spring context; @Mock is plain Mockito (no context).
